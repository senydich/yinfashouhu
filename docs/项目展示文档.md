## 一、技术栈页

### 1.1 核心技术栈概述

银发守护后台管理系统采用 Spring Boot + Thymeleaf + MyBatis + MySQL 的传统 Java Web 技术路线。系统整体按照 Controller、Service、Dao、Mapper、Entity、Template 分层组织代码，后端负责业务处理和数据访问，前端页面通过 Thymeleaf 渲染数据并提供后台管理交互。

| 技术 | 项目版本 | 主要用途 | 项目中的体现 |
| --- | --- | --- | --- |
| Java | 1.8 | 项目基础开发语言 | `pom.xml` 中配置 `source`、`target` 均为 1.8 |
| Spring Boot | 2.6.13 | 快速搭建 Web 应用，管理 Bean、MVC、配置和启动流程 | `YinfashouhuApplication` 启动类，`@Controller`、`@Service`、`@Autowired` 等注解 |
| Spring MVC | 由 Spring Boot 2.6.13 管理 | 接收浏览器请求，完成页面跳转、参数绑定和模型数据传递 | `LoginController`、`FrameController`、`UserController` 等控制器 |
| Thymeleaf | 3.0.15.RELEASE | 服务端模板渲染，将后端 Model 数据展示到 HTML 页面 | `templates` 目录下的 `login.html`、`main.html`、`pages/user-info.html` 等 |
| MyBatis Spring Boot Starter | 2.2.2 | 持久层框架，负责 Java 对象与 SQL 查询之间的映射 | `dao` 接口配合 `resources/mappers/*.xml` 完成数据库操作 |
| MySQL | 5.7.21 | 系统业务数据存储 | 数据库备份文件显示服务端版本为 MySQL 5.7.21，库名为 `yinfashouhu` |
| MySQL Connector/J | 8.0.31 | Java 连接 MySQL 数据库的 JDBC 驱动 | `application.properties` 中配置 `com.mysql.cj.jdbc.Driver` |
| Maven | 依赖管理工具 | 统一管理项目依赖、编译、测试和打包 | `pom.xml` 和 `spring-boot-maven-plugin` |

### 1.2 技术用途说明

Spring Boot 用于统一管理系统启动、Web 请求处理、依赖注入和配置加载。项目通过 `@SpringBootApplication` 启动应用，通过 Controller 接收页面请求，通过 Service 封装业务逻辑，通过 MyBatis Mapper 执行数据库操作。

Thymeleaf 用于后台管理页面渲染。系统中的左侧菜单、用户列表、老人信息、健康数据、阈值设置等页面都位于 `src/main/resources/templates` 目录下，页面中使用 `th:text`、`th:each`、`th:field`、`th:if` 等标签显示后端传入的数据。

MyBatis 用于实现 SQL 与 Java 对象的映射。项目将 SQL 写在 `src/main/resources/mappers` 目录中，例如 `UserMapper.xml` 负责用户列表查询、新增、修改、删除等 SQL，`UserRoleMapper.xml` 负责用户与角色关联关系维护。

MySQL 用于存储系统中的用户、角色、权限、老人档案、健康数据、设备记录、行为报告、操作日志等业务数据。项目数据库连接配置如下：

```properties
spring.datasource.url=jdbc:mysql://localhost:3306/yinfashouhu?useUnicode=true&characterEncoding=utf8&serverTimezone=Asia/Shanghai
spring.datasource.username=root
spring.datasource.driver-class-name=com.mysql.cj.jdbc.Driver
mybatis.mapper-locations=classpath:mappers/*.xml
mybatis.type-aliases-package=com.dtdx.entity
```

### 3.4 关键代码展示

#### 控制器代码

`UserController` 负责接收用户管理相关请求，包括列表、新增、编辑和删除。

```java
@Controller
@RequestMapping("/user")
public class UserController {
    @Autowired
    private UserService userService;

    @Autowired
    private OperationLogService operationLogService;

    @Autowired
    private RoleService roleService;

    @RequestMapping("/userList")
    public String userList(Model model, User user) {
        List<User> userList = userService.userList(user);
        model.addAttribute("userList", userList);
        return "pages/user-info";
    }

    @RequestMapping("/addView")
    public String addView(Model model) {
        model.addAttribute("user", new User());
        model.addAttribute("roleList", roleService.listAllActive());
        model.addAttribute("selectedRoleIds", null);
        model.addAttribute("formAction", "/user/add");
        return "pages/user-add";
    }

    @RequestMapping("/updateView")
    public String updateView(@RequestParam("id") Integer id, Model model) {
        User user = userService.getById(id);
        model.addAttribute("user", user);
        model.addAttribute("roleList", roleService.listAllActive());
        model.addAttribute("selectedRoleIds", userService.findRoleIdsByUserId(id));
        model.addAttribute("formAction", "/user/update");
        return "pages/user-add";
    }
}
```

#### 用户列表动态 SQL

```xml
<select id="userList" resultType="User">
    select
    <include refid="BaseColumnList"/>
    from t_user
    <where>
        <if test="loginName != null and loginName != ''">
            login_name like concat('%', #{loginName}, '%')
        </if>
        <if test="realName != null and realName != ''">
            or real_name like concat('%', #{realName}, '%')
        </if>
        <if test="phone != null and phone != ''">
            or phone like concat('%', #{phone}, '%')
        </if>
    </where>
    order by id desc
</select>
```

## 五、开发过程页

### 5.1 开发流程

本系统开发过程按照“数据库设计 -> 实体类与 Mapper 编写 -> Service 业务封装 -> Controller 请求处理 -> Thymeleaf 页面渲染 -> 功能测试与截图整理”的顺序推进。

1. 数据库设计：根据养老管理业务划分用户、角色、权限、老人、健康数据、设备、行为记录、日志等数据表。
2. 后端分层开发：每个业务模块基本对应 Entity、Dao、Mapper、Service、Controller，保证代码职责清晰。
3. 前端页面开发：使用 Thymeleaf 编写后台管理页面，通过左侧菜单切换右侧内容区。
4. 权限与菜单联动：登录后把权限地址保存到 Session，左侧菜单根据权限变量控制显示。
5. 功能验证：对登录、列表查询、新增、编辑、删除、搜索筛选等后台常用流程进行验证，并整理截图。

### 5.2 典型技术难点与解决思路

| 技术难点 | 问题表现 | 解决思路 |
| --- | --- | --- |
| 左侧菜单需要根据权限动态显示 | 不同角色登录后应看到不同菜单，如果全部写死会导致权限控制不准确。 | 登录成功后查询当前用户的角色和权限地址，保存到 Session；`FrameController` 根据权限生成 `canUser`、`canRole`、`canHealthData` 等布尔值；`left.html` 使用 `th:if` 控制菜单显示。 |
| 页面访问需要鉴权 | 用户未登录或无权限时仍可能直接输入 URL 访问后台页面。 | 编写 `AuthInterceptor` 拦截请求；登录页和静态资源放行；普通页面先判断 Session 登录状态，再判断权限地址是否匹配，不符合条件时跳转到无权限页面。 |
| 新增和编辑表单重复度高 | 用户新增和编辑字段基本一致，如果写两个页面会造成维护成本增加。 | 复用 `user-add.html` 表单模板，通过 `formAction` 区分新增和编辑提交地址，通过 `th:object`、`th:field` 自动完成数据绑定和回显。 |
| 用户与角色是多对多关系 | 保存用户时不仅要保存用户基础信息，还要维护用户角色关联表。 | 新增用户后使用 `useGeneratedKeys` 获取用户 ID，再写入 `t_user_role`；编辑时先删除旧角色关联，再批量写入新角色关联。 |
| 删除用户可能留下关联脏数据 | 如果只删除 `t_user`，`t_user_role` 中会保留无效用户 ID。 | 在 `UserService.delete` 中先执行 `userRoleDao.deleteByUserId(id)`，再执行 `userDao.delete(id)`，保证关联数据同步清理。 |
| 模糊搜索需要支持多个字段 | 用户希望一个搜索框同时按登录名、姓名、电话查询。 | 前端将关键字同时传给 `loginName`、`realName`、`phone`；MyBatis 使用动态 SQL 拼接 `like` 条件，`<where>` 标签自动处理开头的 `or`。 |
| 中文页面和数据库编码问题 | 页面、请求参数或数据库内容可能出现乱码。 | 项目统一配置 UTF-8 编码，`spring.thymeleaf.encoding=UTF-8`、`server.servlet.encoding.force=true`，数据库连接使用 `useUnicode=true&characterEncoding=utf8`。 |

### 5.3 权限控制实现思路

登录时，系统会校验用户账号、密码和启用状态。登录成功后，系统把当前用户、角色名称、角色编码和权限 URL 写入 Session：

```java
request.getSession().setAttribute("loginUser", user);
request.getSession().setAttribute("permissionUrls", roleService.getPermissionUrlsByUserId(user.getId()));
request.getSession().setAttribute("roleNames", roleService.getRoleNamesByUserId(user.getId()));
request.getSession().setAttribute("roleCodes", roleService.getRoleCodesByUserId(user.getId()));
```

页面加载左侧菜单时，`FrameController` 根据 Session 中的权限地址判断当前用户是否能访问某个模块，并将判断结果传给模板：

```java
model.addAttribute("canUser", canAccess(superAdmin, permissionUrls, "/user/userList"));
model.addAttribute("canRole", canAccess(superAdmin, permissionUrls, "/role/roleList"));
model.addAttribute("canThreshold", canAccess(superAdmin, permissionUrls, "/hdt/getOne"));
```

左侧菜单模板再根据变量决定是否显示对应菜单：

```html
<button class="nav-item" type="button"
        data-page="userMgmt"
        data-url="/user/userList"
        th:if="${canUser}">
    <i class="fas fa-users-cog"></i><span>用户管理</span>
</button>
```
